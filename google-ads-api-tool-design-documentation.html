Google Ads API Tool Design Documentation

Tool name: Ikonoverde Internal Marketing Assistant

Company: Ikonoverde

Google Ads manager account ID: 525-687-9092

Primary use: Internal keyword planning, reporting, campaign creation, and campaign management for Ikonoverde-owned advertising accounts.

1. Product Overview

The Ikonoverde Internal Marketing Assistant is an internal Laravel-based marketing tool used by authorized Ikonoverde staff to support SEO and paid search planning for Ikonoverde's ecommerce business. The tool helps staff research search demand, evaluate keyword opportunities, review advertising performance, and prepare advertising campaigns for products sold by Ikonoverde.

The tool is not offered to third parties, external customers, agencies, resellers, or the general public. It is used only by authorized internal personnel for Google Ads accounts owned or controlled by Ikonoverde.

2. Google Ads API Capabilities

The tool provides or is intended to provide the following Google Ads API capabilities:

  • Keyword Planning Services
  • Reporting
  • Campaign Creation
  • Campaign Management

The initial implementation focuses on Keyword Planner keyword ideas, average monthly search volume, competition, and CPC bid range data. Planned near-term capabilities include creating and managing Search campaigns, followed by support for Performance Max and Display campaign workflows where appropriate for Ikonoverde's ecommerce advertising strategy.

3. Supported Campaign Types

The tool is designed to support the following campaign types:

  • Search
  • Performance Max
  • Display

Search campaign support is the first priority because the current tool focuses on keyword research and keyword-based campaign planning. Performance Max and Display support are planned for future internal campaign creation and management workflows.

4. Intended Users

The tool is used only by authorized Ikonoverde staff members responsible for marketing, ecommerce, analytics, and advertising operations. Users must authenticate into the Ikonoverde application before accessing marketing assistant functionality.

5. Account Scope

The Google Ads API integration is limited to Ikonoverde-owned Google Ads accounts and manager/client accounts controlled by Ikonoverde. The tool is not intended to manage advertising accounts for unrelated third parties.

6. Current API Workflow

The current keyword planning workflow is:

  • An authorized internal user enters a seed keyword, product topic, product category, country, language, and result limit.
  • The application sends a Google Ads API Keyword Planner request using the configured Google Ads customer account.
  • The application receives keyword ideas and metrics from Google Ads.
  • The application normalizes the response into keyword, average monthly searches, competition, competition index, bid range, and monthly volume fields.
  • The normalized response is displayed to the internal user for planning and analysis.

7. Planned Campaign Creation Workflow

The planned campaign creation workflow is:

  • An authorized internal user reviews keyword and campaign recommendations inside the application.
  • The user confirms campaign settings, budgets, targeting, keywords, ad groups, ads, and assets before any write operation is performed.
  • The application creates or updates Google Ads entities only after internal user approval.
  • The application records the result of API operations for internal audit and troubleshooting.

The tool will not automatically create or modify campaigns without an authorized internal user action.

8. Data Accessed

The tool may access the following Google Ads data:

  • Keyword ideas and keyword planning metrics.
  • Average monthly searches, competition, and CPC bid range metrics.
  • Campaign, ad group, keyword, ad, asset, budget, and targeting configuration.
  • Advertising performance and reporting data for Ikonoverde-owned accounts.

9. Data Storage and Retention

Keyword research and campaign planning outputs may be displayed inside the internal application and may be stored in application logs or database records for internal analysis, debugging, and audit purposes. Google Ads OAuth credentials, developer tokens, and other secrets are stored server-side in environment variables or secure deployment configuration and are not exposed to browser clients or unauthenticated users.

Google Ads data is not sold, resold, published, or shared externally. It is used only to support Ikonoverde's own marketing and advertising operations.

10. Security Controls

  • The Google Ads API integration is available only to authenticated internal users.
  • OAuth credentials are generated by a Google user with authorized access to the Google Ads account.
  • Developer tokens, client secrets, and refresh tokens are stored server-side only.
  • The application does not expose Google Ads credentials to users or third-party clients.
  • Future write operations for campaign creation and management will require explicit internal user approval.

11. Compliance and Policy Use

The tool will be used in accordance with Google Ads policies. It will not scrape, resell, redistribute, or publicly expose Google Ads data. Campaign creation and management workflows will be used only for Ikonoverde-owned advertising accounts and will be reviewed by authorized staff before launch or modification.

12. Third-Party Access

The tool is not a software-as-a-service product, agency dashboard, public API, or third-party reporting product. No external customers can use the tool to access or manage Google Ads accounts.

13. Technical Implementation Summary

The application is built with Laravel and uses the Google Ads REST API. Authentication uses OAuth 2.0 with the Google Ads scope https://www.googleapis.com/auth/adwords. The application stores Google Ads configuration in server-side environment variables, including developer token, customer ID, OAuth client ID, OAuth client secret, and refresh token.
